# GIT
Git é um software de versionamento de versão, ou seja auxilia equipes a gerenciar e acompanhar alterações de um projeto, permitindo que trabalhem simultaneamente e facilitando o fluxo de trabalho.
O fluxo de trabalho com Git começa com a criação de branches para cada conjunto de alterações, permitindo que desenvolvedores trabalhem em paralelo. Após implementar mudanças, usa-se `git add` para preparar arquivos para commit, `git commit` para registrar alterações localmente, e `git push` para enviá-las ao repositório remoto. A sincronização de branches e a realização de merges são etapas cruciais para integrar alterações.

## [[commit]]
Armazena o estado atual dos arquivos rastreados, permitindo uma monitoração das versões mais simples. Os arquivos que vão ser alterados no commit são de escolha do desenvolvedor, esses arquivos são chamados de Staging area. Ao fazer o commit é recomendado escrever uma mensagem breve, especificando alguma alteração ou a versão do código
