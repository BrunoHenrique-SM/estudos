o git é um sistema de controle de versão, em que é possível armazenar as versões do código. Quando iniciamos um novo diretório, começamos uma branch(ramo), como o nome diz, é possível criar diversas ramificações partindo de um mesmo código fonte

## [[commit]]
Armazena o estado atual dos arquivos rastreados, permitindo uma monitoração das versões mais simples. Os arquivos que vão ser alterados no commit são de escolha do desenvolvedor, esses arquivos são chamados de Staging area. Ao fazer o commit é recomendado escrever uma mensagem breve, especificando alguma alteração ou a versão do código
