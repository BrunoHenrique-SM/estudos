Armazena o estado atual dos arquivos rastreados, permitindo uma monitoração das versões. Os arquivos que vão ser alterados no commit são de escolha do desenvolvedor, esses arquivos são chamados de Staging area. Ao fazer o commit é recomendado escrever uma mensagem breve, especificando a feature que foi alterada, então é feito o checkout da branch e depois a merge, para que seja acoplado ao main
