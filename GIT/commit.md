# commit
Armazena o estado atual dos arquivos rastreados, permitindo uma monitoração das versões. Os arquivos que vão ser alterados no commit são de escolha do desenvolvedor, esses arquivos são chamados de [[Staging]] area. Ao fazer o commit é recomendado escrever uma mensagem breve, especificando a feature que foi alterada. Os commits ocorrem na máquina e com o [[push]] são enviados para o [repositório](Repositório)
